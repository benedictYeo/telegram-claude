  <action>
Create `src/db/index.ts` with:

1. Import `Database` from `better-sqlite3` and `path` from `path`.
2. Resolve DB path: `const DB_PATH = process.env.DB_PATH || path.resolve("data/bot.db");`
   - `process.env.DB_PATH` allows test override to `:memory:` (Pitfall 6 from RESEARCH.md).
3. Module-level `let _db: Database.Database | null = null;` for lazy singleton.
4. Export `getDb()` function:
   - If `_db` is null, create `new Database(DB_PATH)`.
   - Set `db.pragma("journal_mode = WAL")` (Pitfall 5 from RESEARCH.md).
   - Call `initSchema(db)`.
   - Cache in `_db`.
   - Return `_db`.
5. Private `initSchema(db)` function that runs `db.exec()` with:
   - `conversations` table: `id INTEGER PRIMARY KEY AUTOINCREMENT`, `chat_id INTEGER NOT NULL`, `created_at INTEGER NOT NULL`, `archived INTEGER NOT NULL DEFAULT 0`.
   - `messages` table: `id INTEGER PRIMARY KEY AUTOINCREMENT`, `conversation_id INTEGER NOT NULL REFERENCES conversations(id)`, `role TEXT NOT NULL CHECK (role IN ('user', 'assistant'))`, `content TEXT NOT NULL`, `input_tokens INTEGER`, `output_tokens INTEGER`, `created_at INTEGER NOT NULL`.
   - `daily_usage` table: `date TEXT NOT NULL`, `input_tokens INTEGER NOT NULL DEFAULT 0`, `output_tokens INTEGER NOT NULL DEFAULT 0`, `PRIMARY KEY (date)`.
   - Index: `CREATE INDEX IF NOT EXISTS idx_messages_conversation ON messages(conversation_id)`.
   - Index: `CREATE INDEX IF NOT EXISTS idx_conversations_chat ON conversations(chat_id, archived)`.
6. Export `closeDb()` function for test cleanup: if `_db` is not null, call `_db.close()` and set `_db = null`.

Follow project conventions: double quotes, 2-space indent, semicolons, `.js` extension on imports, named exports only.
